# HiAir MVP Spec (на основе PDF-плана)

## Цель MVP

Построить мобильное приложение, которое дает персональные рекомендации по активности на улице
в условиях жары и загрязнения воздуха.

## Входит в MVP (Phase 2 из PDF)

- Онбординг:
  - выбор типа профиля (adult, child, elderly, asthma, allergy, runner),
  - геолокация,
  - вопросы чувствительности.
- Главный экран:
  - текущий уровень риска,
  - персональные рекомендации.
- Daily planner:
  - почасовой риск,
  - безопасные окна для улицы/проветривания.
- Symptom log:
  - быстрые отметки (кашель, свистящее дыхание, головная боль, усталость, качество сна).
- Настройки:
  - профили, уведомления, разрешения.
- Push-оповещения:
  - предупреждения по порогам риска.

## Не входит в MVP (после запуска)

- HealthKit / Health Connect.
- Премиум-монетизация и платные ограничения фич (paywall).
- ML-предиктивные модели.
- Социальные и B2B-функции.

## Уточнение по подпискам в текущем коде

В текущем репозитории уже есть технический контур `subscriptions` (планы, статус, webhook, cancel/activate)
для beta-проверок контракта и ownership/security сценариев.

Это **не** означает готовую коммерческую paywall-монетизацию MVP:
- monetization UX, billing compliance и store-финализация остаются пост-MVP задачами.

## Персоны

- Родители с детьми.
- Пожилые пользователи.
- Люди с астмой/аллергией.
- Outdoor-спортсмены.
- Outdoor-работники.
- Жители городов, следящие за здоровьем.

## KPI MVP

- Конверсия онбординга >= 60%.
- Retention D1 >= 35%.
- Логи симптомов (1+ раз в неделю) >= 30% MAU.
- Open rate push >= 10%.

## Ограничения и юридический контур

- Позиционирование как wellness guidance, а не medical advice.
- Согласие на обработку данных и удаление данных по запросу (GDPR/CCPA).
